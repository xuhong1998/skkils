## 标签提取系统架构

### 核心组件

1. **AI标签提取提示词**
   - 详细的标签分类体系
   - 提取标准和规范
   - 输出格式要求
   - 质量评估标准

2. **标签块格式标准**
   - 标准化Markdown格式
   - 符合Obsidian识别要求
   - 分类清晰的标签区块
   - 易于维护和更新

3. **集成方式**
   - 自动触发（写日记时自动执行）
   - 手动触发（独立标签提取功能）
   - 批量处理（历史数据批量提取）

### 标签分类体系

**主要标签（3-5个）：**
- 投资理财、数字成瘾、家庭关系、学习成长、生活日常、工作职业

**子标签（2-3个/主标签）：**
- 基金交易、抖音使用、家庭冲突、阅读完成、运动健康等

**情绪标签（2-3个）：**
- 正向（开心、满足）、负向（焦虑、烦躁）、中性（平静、思考）

**地点标签（1-2个）：**
- 城市（武汉）、场所（家中、公园、公司）

**人物标签（可选）：**
- 家人（爸爸、妈妈）、朋友、同事

## Obsidian集成特性

### 关系图谱显示

**节点类型：**
- 📄 日记节点：显示日期和标题
- 🏷️ 标签节点：显示标签名称

**连线关系：**
- 日记节点 ↔ 标签节点（双向连接）
- 相同标签的所有日记自动关联

**可视化效果：**
```
        #生活日常
         ↗  ↖
    03-01    03-02
         ↘  ↙
      #运动健康
```

### 配置优化

建议的Obsidian配置：
```json
{
  "showTags": true,
  "tagNodeSize": "usage_weighted",
  "colorGroups": [
    {
      "query": "investment",
      "color": "#FF6B6B"
    },
    {
      "query": "digital_addiction",
      "color": "#4ECDC4"
    },
    {
      "query": "family",
      "color": "#45B7D1"
    }
  ]
}
```

## 技术特性

### AI驱动 vs Python代码

**优势：**
1. **语义理解强**：AI能理解隐含关系和上下文
2. **开发简单**：无需编写复杂的Python代码
3. **维护容易**：通过提示词调整提取规则
4. **灵活性强**：快速响应新的标签需求

**实现方式：**
- 使用精心设计的AI提示词
- 标准化的输出格式要求
- 零代码的标签提取逻辑
- 最小化的辅助脚本（仅用于格式化）

### 标签块格式设计

**标准格式：**
```markdown
---
## 🏷️ AI提取标签

### 主要标签
#标签1 #标签2 #标签3

### 子标签  
#子标签1 #子标签2

### 情绪标签
#情绪1 #情绪2

### 地点标签
#地点1

### 人物标签
#人物1

---
```

**格式要求：**
- 必须以`---`开始和结束
- 标题必须是`## 🏷️ AI提取标签`
- 标签必须以`#`开头
- 标签之间用空格分隔

### 性能表现

| 操作 | 时间 | 说明 |

| 单篇日记标签提取 | 30秒-2分钟 | 取决于内容复杂度 |
| 批量处理27篇日记 | 约15-25分钟 | 并行处理可优化 |
| 后台标签提取 | 10-15秒 | 用户无感知 |

## 风险和挑战

### 技术风险

1. **AI提取一致性**
   - 风险：不同时间提取可能有差异
   - 缓解：详细的提取标准和示例

2. **性能和延迟**
   - 风险：AI处理可能有延迟
   - 缓解：异步处理和进度提示

### 使用风险

1. **标签质量**
   - 风险：提取的标签可能不准确
   - 缓解：用户确认和调整机制

2. **标签过度提取**
   - 风险：可能提取过多标签
   - 缓解：严格控制提取数量和标准
