# diary-scatter-organizer 更新总结 - 想法功能

## 更新日期

2026-03-02

---

## 问题描述

原来的 `diary-scatter-organizer` skill 只解析时间戳记录（`HH:MM 内容`），会忽略"想法"部分。

**问题示例：**

```
8:30  早上吃了碗热干面
12:00 中午吃千张肉丝
18:00 下班，今天下了一个早班...
19:00 在家自己做饭，还是吃豆丝炒腊肠

想法
今天同时叫我下下去香港麦理浩径徒步...
```

"想法"部分被忽略，没有记录到日记中。

---

## 解决方案

添加"想法"识别功能，支持多种关键词，并将想法内容填充到日记的"感悟"部分。

---

## 实现内容

### 1. 新增函数 `parse_input()`

**功能：** 解析输入，分离时间戳记录和想法

**返回值：**
```python
{
    'records': [(时间, 内容), ...],
    'ideas': '想法内容（或空字符串）'
}
```

**支持的想法关键词：**
- `想法`（无冒号或带冒号）
- `感悟`（无冒号或带冒号）
- `思考`（无冒号或带冒号）
- `感想`（无冒号或带冒号）

### 2. 修改函数 `generate_template()`

**新增参数：**
```python
def generate_template(
    ...
    ideas: str = "",  # 新增想法参数
) -> str:
```

**功能：**
- 如果有想法内容，填充到"感悟"部分
- 否则保持空引用块（`>`）

### 3. 更新文档 `SKILL.md`

**新增内容：**
- 核心原则中添加"想法提取"
- 完整流程中添加想法提取步骤
- 添加想法解析规则和示例
- 更新生成模版框架说明
- 添加带想法的完整示例

## 使用示例

### 输入

```
8:30  早上吃了碗热干面
12:00 中午吃千张肉丝
18:00 下班，今天下了一个早班，感觉还不错。回到家后发现天还没黑下来，有出门去了一趟公园散步
19:00 在家自己做饭，还是吃豆丝炒腊肠
20:00 本来想继续写日历app的，但是有刷抖音去了，到了20：40才开始写日历app，让ai把前后端都生成出来，但是还是有很多问题

想法
今天同时叫我下下去香港麦理浩径徒步，我看他坐卧铺去深圳然后去香港，看成本很低我也很想去
```

### 输出（感悟部分）

```markdown
## 感悟

今天同时叫我下下去香港麦理浩径徒步，我看他坐卧铺去深圳然后去香港，看成本很低我也很想去

---
```
